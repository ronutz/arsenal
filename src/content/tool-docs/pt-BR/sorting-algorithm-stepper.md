O passo a passo recebe uma lista de 2 a 16 números e executa a estratégia escolhida uma decisão visível por vez. Cada passo mostra o estado do array com os índices tocados destacados, o tipo do passo (comparação, troca, escrita, nota) e uma linha de motivo para o movimento - a invariante que o algoritmo está protegendo. Os contadores no final são o ponto: rode a mesma lista em várias estratégias, ou aumente a lista, e observe como comparações e escritas escalam. Essa curva de escala é o Big-O tornado concreto, como [o primer de algoritmos](/learn/what-is-an-algorithm) desenvolve.

Dois comportamentos didáticos são preservados de propósito. O bubble sort carrega a clássica saída antecipada: entregue uma lista já ordenada e ele detecta o passe limpo e para após uma varredura linear. O quicksort usa a forma comum de ensino - partição de Lomuto com o último elemento como pivô - o que significa que uma lista já ordenada o empurra para o pior caso O(n²); cole uma e veja os contadores explodirem. Tudo roda localmente, a saída passa por checagem de invariante de ordenação a cada execução, e o mesmo motor é chamável pela API do site com um corpo JSON de `{"text": "5 3 8 1", "strategy": "merge"}`.
